# Matriz de competencias - Android

## Objetivo
Asegurar progresion verificable desde nivel cero hasta maestria con evidencia ejecutable y defensa tecnica.

## Niveles y evidencias

| Nivel | Capacidad objetivo | Evidencia minima obligatoria |
| --- | --- | --- |
| Core Mobile + Nivel Cero | Comprender capas, invariantes y estructura base | Diagrama de capas + glosario de flechas + snippet compilable |
| Junior | Construir feature completa con arquitectura limpia | Caso de uso + ViewModel + test unitario en verde |
| Midlevel | Integrar offline-first, contratos y sincronizacion | Test de integracion + diagrama de wiring + trade-off escrito |
| Senior | Operar modulos con observabilidad y governance | Metricas base + runbook corto + decision de riesgo |
| Maestria | Defender arquitectura enterprise en evolucion | ADR + estrategia de rollout/rollback + checklist release |
| Proyecto Final | Entregar sistema defendible end-to-end | Demo reproducible + evidencia tecnica + defensa oral |

## Criterio de paso
1. No se aprueba nivel sin evidencia tecnica y defensa breve.
2. Si falla un gate, el nivel queda en recuperacion.
3. Toda evidencia debe poder ejecutarse desde repo.

## Reglas de evaluacion
1. Comprension: explica limites y dependencias.
2. Aplicacion: demuestra con codigo y pruebas.
3. Defensa: argumenta decision y alternativa descartada.
4. Operacion: valida observabilidad y resiliencia.
